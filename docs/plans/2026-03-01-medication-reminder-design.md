# 用药提醒 Web 应用 — 设计文档

## 核心定位

药剂剩余次数倒计时 Web 应用。解决一个问题：**永远不要再用到空的吸雾剂**。

## 用户背景

- 布地奈德吸雾剂，治疗哮喘（3 年病史，目前稳定）
- 当前处方：早 1 次 + 晚 2 次 = 每日 3 次
- 每支药剂 60 次（约 20 天用完）
- 医生会根据复诊情况调整每日用量
- 曾因使用空药剂导致严重不适

## 核心功能

### 1. 剩余次数倒计时

默认每天按设定的每日次数自动扣减，无需手动打卡。

核心公式：
- 已用次数 = 各时段（每日次数 × 天数）之和
- 剩余次数 = 总次数 - 已用次数
- 剩余天数 = 剩余次数 / 当前每日次数

### 2. 分段用量记录

每日次数支持中途修改，按时间线分段计算：
- 每次修改生成一条记录：`{ date, dailyDoses }`
- 计算已用次数时，按各段分别计算再求和
- 示例：3/1 设定 2 次/天 → 3/16 改为 3 次/天，系统精确计算

### 3. 分级换药提醒

- **提前约一周**（剩余 ≤ 21 次）：普通提醒 —"该准备新药了"
- **最后一天**（剩余 ≤ 当前每日次数）：紧急提醒 —"明天就用完了"
- 使用 Web Notifications API（需用户授权）

### 4. 可配置参数

- **总次数**：新药默认 60 次，可修改
- **每日次数**：默认 3 次，可修改（修改后从当天生效）
- **剩余次数**：可手动修正

## 页面结构

单页面，从上到下三个区域：

### 状态区（视觉核心）
- 环形进度条，可视化剩余比例
- 大字显示：剩余 **42** 次 / 共 60 次
- 已用 **18** 次
- 每日 **3** 次
- 预计还能用 **14** 天（3月15日用完）

### 操作区
- **开新药**按钮 — 弹出输入框，默认 60 次，可修改总次数
- **修正次数**按钮 — 手动调整当前剩余次数
- **调整每日次数**按钮 — 修改后从当天生效

### 历史区（底部折叠）
- 每支药的使用周期记录列表（启用日期、总次数、用完日期）
- 为后续复诊功能预留

## 数据模型

```typescript
interface Cartridge {
  id: string;
  totalDoses: number;        // 总次数（如 60）
  startDate: string;         // 启用日期
  endDate?: string;          // 用完/换药日期
  manualAdjustments: Array<{ // 手动修正记录
    date: string;
    remainingDoses: number;
    reason?: string;
  }>;
  dosageChanges: Array<{     // 每日次数变更时间线
    date: string;
    dailyDoses: number;
  }>;
}

interface AppState {
  currentCartridge: Cartridge | null;
  history: Cartridge[];
  notificationEnabled: boolean;
}
```

## 剩余次数计算逻辑

```
对于当前药剂：
  1. 从 startDate 开始，按 dosageChanges 时间线分段
  2. 每段：该段天数 × 该段每日次数 = 该段已用次数
  3. 总已用 = 各段已用之和
  4. 若有 manualAdjustments，以最近一次修正为基准重新计算
  5. 剩余次数 = 总次数 - 总已用
  6. 剩余天数 = 剩余次数 / 当前每日次数（向下取整）
```
